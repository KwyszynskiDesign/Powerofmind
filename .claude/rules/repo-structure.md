# repo-structure

- Pliki strony żyją w `site/`.
- Dokumentacja planów i audytów w `docs/`.
- Nie mieszaj plików źródłowych z dokumentacją.
- Nowe podstrony dodawaj w `site/stories/` zgodnie z konwencją `site/stories/{slug}/index.html`.
- Powtarzalne fragmenty HTML trzymaj w `site/components/`.
- CSS rozdziel na: variables, base, layout, components, utilities.
